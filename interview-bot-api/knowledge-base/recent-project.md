# Recent Project — Keen (Ingenio)

## Overview
Client/Company: Ingenio
Duration: 2020 - Present (6+ years)
My Role: Lead Software Engineer / Team Lead
Team Size: 20+ Cross-functional team including designers, product managers, QE and developers

## What Is The Product
Keen is a consumer platform that connects users with psychics, tarot card readers, astrologers, and other spiritual advisors for online chat or phone consultations. The platform handles real-time advisor matching, session management, billing, loyalty rewards, and marketing campaigns at scale. It is an e-commerce style platform with high availability requirements.

## What I Built / What I Worked / Recent Projects
- Semantic Search
- JWT AUTH Migration from existing Session based AUTH 
- Feedback Search
- Third Party Partner inegration

## Tech Stack
- Backend: .NET / .NET Core, C#, Web API, REST
- Frontend: React, Next.js, TypeScript
- Database: SQL Server / PostGres SQL
- Cloud: Microsoft Azure
- CI/CD: TeamCity, Azure DevOps
- Source Control: GitHub
- Logging: Splunk (centralized logging)
- Analytics: Mixpanel (event tracking)
- Support: Zendesk
- Loyalty: Zinrelo (loyalty rewards platform)
- Marketing: Iterable (marketing campaigns)
- CMS: Content Stack

## My Key Contributions
1. Architected and designed scalable software solutions ensuring reliability and maintainability
2. Led the frontend and backend development across the platform
3. Integrated third-party platforms — Zinrelo for loyalty rewards, Iterable for marketing, Zendesk for support

## The Biggest Challenge I Faced
Integrating multiple third-party platforms (Zinrelo, Iterable, Zendesk, Content Stack) into a
cohesive system while maintaining performance and reliability. Each platform had different APIs
and data models, requiring careful orchestration and robust error handling.

## What I Am Most Proud Of
Growing into the Lead Software Engineer role and building a platform that connects thousands of
users with advisors, while successfully integrating a full ecosystem of third-party tools that
power loyalty, marketing, and support — all running reliably on Azure.

## What I Would Do Differently
I would introduce automated integration testing for third-party services earlier in the development
cycle, and invest in better observability tooling from the start to reduce debugging time across
the distributed integrations.
